Guia Introdutório ao Docker

O que é Docker?

Docker é uma plataforma aberta para desenvolver, enviar e executar aplicações em containers. Containers permitem empacotar uma aplicação com todas as suas dependências em uma unidade padronizada para desenvolvimento e deploy.

Analogia Simplificada

Imagine containers como malas de viagem que contêm tudo o que seu aplicativo precisa para funcionar (código, bibliotecas, configurações), permitindo que ele seja "transportado" e executado em qualquer ambiente (desenvolvimento, teste, produção) sem problemas de compatibilidade.

Conceitos Fundamentais

1. Container

Um container é uma unidade de software leve, autossuficiente e executável que inclui tudo o que é necessário para executar um aplicativo: código, runtime, ferramentas do sistema, bibliotecas e configurações.

2. Imagem Docker

Uma imagem é um modelo somente leitura com instruções para criar um container. Ela contém o sistema de arquivos e os metadados necessários para executar a aplicação.

docker pull nginx # Baixa a imagem oficial do Nginx

3. Dockerfile

Um Dockerfile é um arquivo de texto que contém todas as instruções necessárias para construir uma imagem Docker.

Exemplo básico:

# Use the official Nginx image as the base image
FROM nginx:latest

# Copy the custom nginx.conf to the container
COPY nginx.conf /etc/nginx/nginx.conf

# Copy the HTML file to the default Nginx HTML directory
COPY index.html /usr/share/nginx/html/

# Expose port 80
EXPOSE 80

# Start Nginx
CMD ["nginx", "-g", "daemon off;"]

4. Docker Hub/Registry

Repositórios onde imagens Docker são armazenadas e compartilhadas. O Docker Hub é o registry público oficial, mas você também pode criar registries privados.

Vantagens do Docker

  • Isolamento: Aplicações rodam em ambientes isolados
  • Portabilidade: "Funciona na minha máquina" deixa de ser problema
  • Eficiência: Consome menos recursos que máquinas virtuais
  • Consistência: Ambientes idênticos em desenvolvimento e produção
  • Escalabilidade: Fácil replicação e orquestração de containers
  • Versionamento: Controle de versões de imagens como código
  • Rápida implantação: Containers iniciam em segundos
  • Ecossistema: Grande comunidade e ferramentas complementares

Exemplo Prático

Para executar um servidor web Nginx em um container Docker:

# Baixar a imagem oficial do Nginx
docker pull nginx

# Executar um container baseado na imagem
docker run --name meu-nginx -d -p 8080:80 nginx

# Acessar o servidor em http://localhost:8080

Este comando cria um container isolado executando o Nginx, mapeando a porta 80 do container para a porta 8080 da sua máquina.

Quando usar Docker?

  • Padronizar ambientes de desenvolvimento entre equipes
  • Simplificar configuração de ambientes complexos
  • Implementar microsserviços
  • Criar pipelines CI/CD eficientes
  • Implementar deploy escalável em produção
  • Executar múltiplas versões do mesmo software

Docker revolucionou a forma como desenvolvemos, entregamos e executamos aplicações modernas, sendo hoje uma habilidade essencial para desenvolvedores e administradores de sistemas.
